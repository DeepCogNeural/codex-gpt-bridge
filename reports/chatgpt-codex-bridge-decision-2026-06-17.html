codex-gpt-bridge · decision memo · 2026-06-17

ChatGPT 到 Codex 本地桥接决策报告

目标：让 ChatGPT/GPTs 做规划和调研，通过 MCP 调用本机受控 Codex 执行器。MCP 指 Model Context Protocol，即模型调用外部工具的标准协议。
TL;DR: 结论是新建小型 MIT 项目，不 fork ColaMeta。原因很简单：本机 Codex 已经有官方 codex mcp-server，我们只需要补一个 ChatGPT 可访问的 Streamable HTTP MCP 网关，并把启动目录、认证、session 和 sandbox 边界做窄。

01

Decision

本节给最终路线，不展开所有备选细节。

Adopt Now

发布 codex-gpt-bridge：TypeScript、MIT、只暴露 bridge_status、codex_run、codex_reply 三个工具。

Hold Back

不直接 fork ColaMeta，不暴露原始 shell，不把 codex app-server 全量挂给 ChatGPT。

Revisit Later

后续可加 preview/apply、OAuth、任务队列和 Web console；首版先验证真实桥接价值。

02

Evidence

关键证据来自官方文档、本机 Codex CLI、候选 GitHub 项目和本项目验证。

候选结论理由
riccilnl/colameta高度相关，但不 fork它覆盖 GPTs 到本地执行器、plan、preview/apply、Git 闭环；但 Alpha、体量大，且许可文本禁止商业使用。
tuannvm/codex-mcp-server参考，不作为基础成熟度较好，但目标是 Claude/编辑器调用 Codex，README 仍偏 API key/Claude 工作流，不是 ChatGPT HTTP MCP 安全网关。
playcations/FileSystem-MCP-for-GPT反例/参考说明 ChatGPT 接本机文件系统可行，但暴露 shell、apply_patch、文件删除等能力，首版不应这么宽。
官方 codex mcp-server采用本机实测暴露 codex 和 codex-reply，正好是我们需要的执行核心。
Source notes 
OpenAI Apps SDK quickstart: ChatGPT app needs an MCP server that defines tools and exposes capabilities to ChatGPT.
OpenAI Connect from ChatGPT docs: MCP server must be HTTPS reachable; local development can use Secure MCP Tunnel, ngrok, or Cloudflare Tunnel.
OpenAI Codex MCP docs: Codex supports MCP servers; local stdio and streamable HTTP are documented.
OpenAI Secure MCP Tunnel docs: private local MCP servers can be reached without public inbound ports.
Local probe: codex-cli 0.134.0; codex mcp-server tools/list returned codex and codex-reply.
GitHub probe: riccilnl/colameta is public, Python, 9 stars at check time, no standard license metadata, README says non-commercial.
03

Implementation

首版代码已经按最小可验证路线实现。

部分文件作用
配置与安全边界src/config.ts解析 host、token、allowed roots、sandbox 和 write 开关；要求 token 或显式本地 no-auth。
上游 Codex 连接src/upstream.ts用 MCP SDK 启动并连接官方 codex mcp-server。
工具层src/tools.ts注册 bridge_status、codex_run、codex_reply，并在转发前做 cwd、sandbox、session 来源和敏感文件预检。
Session 记录src/sessionRegistry.ts只允许继续本 bridge 创建过的 Codex thread。
HTTP MCP 与文档src/server.ts / docs/提供 /mcp、/healthz，并说明 ChatGPT 连接和安全模型。

05

Limits

这些是首版刻意不解决或仍需用户确认的点。

Remaining Risk

没有在真实 ChatGPT UI 里完成连接，因为这需要你账号侧的 Apps/GPTs developer settings 或 Secure MCP Tunnel 配置。首版也没有实现 preview/apply、OAuth、自建 Web console 或任务队列。CODEX_GPT_BRIDGE_ROOTS 只是启动目录门禁，不是操作系统级文件隔离；高风险仓库仍应先做 sanitized staging copy 或 Codex permission profile。Session 记录是内存态，重启会清空。
